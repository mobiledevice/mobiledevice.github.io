methodology

How this detector figures out what phone this is

No app, camera, or account needed — just signals your browser already shares with any website you visit.

01

Reads the user agent

Your browser sends a string identifying its OS, version, and browser build on every page load.

02

Queries client hints

On supported Android browsers, a Client Hints API request returns your exact marketing model name.

03

Checks screen data

Resolution and pixel ratio narrow down the model further, especially useful for identifying iPhones.

04

Reads hardware features

Touch points, CPU cores, memory, and connection type fill out the rest of the profile.

good to know

How accurate is this?

Browser-based detection is reliable for most everyday questions, but it has real limits worth knowing about.

⚠️ Where results can be off

  • iPhone models are estimated. Apple's browsers never reveal the exact model, so the tool matches your screen size against known iPhones and may list more than one candidate.
  • Some Android browsers hide the model. Firefox and privacy-focused browsers may not support the Client Hints API, so detection falls back to a generic Android result.
  • Spoofed user agents change the result. Developer tools, some VPN apps, and privacy browsers can deliberately alter what your browser reports.
  • The exact device name is always in Settings. For a guaranteed answer, check Settings > General > About (iPhone) or Settings > About phone (Android).
